AIX (bluesky) configure args:

--with-bitmode=64
--with-pmapi=/usr/lib
--prefix=/home/bluesky/rosinski
CC=cc_r
Had to hardwire PREFIX= in Makefile.inc because configure seemed to ignore --prefix input. Also had to change all occurrences of xlc and xlc_r to cc and cc_r respectively, because bluesky doesn't have xlc or xlc_r compilers, and again configure failed to recognize these args. For whatever reason, library pmapi seems to work in both 64-bit and 32-bit modes.

Important note: there is a bug in PAPI 3.0 that really necessitates using PAPI 3.0.7 to get useful results in threaded mode under AIX. The required code change is in aix.c where a static definition needs to NOT be static.
